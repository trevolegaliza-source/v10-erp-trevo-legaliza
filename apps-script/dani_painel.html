👋 Funcionário sai da Trevo ▶

📝 Novo processo de cliente existente ▶
Nada! Webhook do board já existe. Dani escuta automaticamente assim que o cartão for criado (via formulário ou manual).
📊 Cliente quer relatório de SLA ▶
1. Vai na seção "📥 Relatório SLA" aqui em cima
2. Coloca código do cliente (opcional), datas (opcional), tipo (opcional)
3. Clica "Gerar rascunho no Gmail"
4. Vai na sua caixa Gmail → pasta Rascunhos
5. Confere se tá certo, edita se quiser, ajusta destinatário, envia
🚨 Dani não respondeu o cliente — o que fazer? ▶
Sequência de checagem:
1. Clica "📋 Diagnóstico completo" — confere se DANI_ATIVA está LIGADA
2. Clica "🩺 Heartbeat" — confere se Claude API está respondendo
3. Clica "🔄 Carregar últimas 30" no Log abaixo — vê se webhook chegou e o que aconteceu
4. Se tudo OK mas mesmo assim Dani não agiu, abre o card no Trello e olha as etiquetas (pode ser que ela já agiu mas você não viu)
🔇 Dani está mandando coisa errada — como pausar? ▶
1. Clica "🔴 Desativar Dani (dry-run)" — para imediato
2. Investiga o problema
3. Quando resolvido, "🟢 Ativar Dani" de novo

Em modo dry-run, Dani só loga, não age. Webhooks continuam chegando.
